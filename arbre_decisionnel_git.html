Arbre Décisionnel des Commandes Git

  • Commencer un Nouveau Projet
    Initialisez un nouveau dépôt Git
    • git init
      (Initialiser un dépôt Git vide)
      Crée un nouveau dépôt Git vide
      • Ajouter des Fichiers/Commits
        Ajoutez des fichiers ou enregistrez des modifications
        • git add <file>
          (Vous avez de nouveaux fichiers ou des modifications ?)
          Ajoutez de nouveaux fichiers ou modifications
        • git commit -m "message"
          (Vous avez ajouté des fichiers avec git add ?)
          Enregistrez vos modifications avec un message de commit
      • Pousser vers Remote
        Envoyez vos commits locaux vers un dépôt distant
        • git remote add origin <url>
          (Vous avez un dépôt distant configuré ?)
          Configurez un dépôt distant pour votre projet
        • git push -u origin main
          (Vous voulez envoyer vos commits locaux vers le dépôt distant ?)
          Envoyez vos commits vers le dépôt distant principal
      • Travailler avec des Branches
        Gérez les branches pour différentes versions de votre projet
        • git branch <new-branch>
          (Vous avez besoin d'une nouvelle branche ?)
          Créez une nouvelle branche pour votre projet
        • git checkout <branch>
          (Vous devez changer de branche ?)
          Changez de branche pour travailler sur une autre version
        • git merge <branch-to-merge>
          (Vous devez fusionner des branches ?)
          Fusionnez les modifications d'une branche dans une autre
      • Collaborer avec une Équipe
        Travaillez avec d'autres personnes sur le même projet
        • git clone <repo-url>
          (Vous voulez cloner un dépôt existant ?)
          Clonez un dépôt Git existant
        • git fetch <remote>
          (Vous voulez récupérer les modifications depuis le dépôt distant ?)
          Récupérez les modifications depuis le dépôt distant sans fusionner
        • git pull <remote> <branch>
          (Vous voulez fusionner les modifications du dépôt distant dans votre branche locale ?)
          Récupérez et fusionnez les modifications du dépôt distant
      • Résoudre des Conflits
        Gérez les conflits de fusion dans votre projet
        • git merge <branch-with-conflict>
          (Vous avez des conflits de fusion ?)
          Fusionnez une branche qui a des conflits
        • Résoudre les conflits dans les fichiers
          (Avez-vous résolu les conflits dans les fichiers concernés ?)
          Résolvez les conflits de fusion dans vos fichiers
        • git add <resolved-file>
          (Avez-vous ajouté les fichiers résolus ?)
          Ajoutez les fichiers après avoir résolu les conflits
        • git commit -m "resolved conflicts"
          (Avez-vous commité les résolutions de conflits ?)
          Commitez vos résolutions de conflits
